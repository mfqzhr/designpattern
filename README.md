#### 设计原则
1. 找出应用中可能需要变化之处，把他们独立出来，不要和那些不需要变化的代码混在一起
2. 针对接口编程，而不是针对实现编程
3. 多用组合，少用继承
#### 单例模式
* 饿汉模式:   
   类加载到内存后,就实例化一个单例,JVM保证线程安全   
   简单实用,推荐使用   
   唯一缺点: 不管用到与否,类加载时就完成实例化   
* 懒汉模式:   
   虽然带来了按需求初始化的目的,但是却带了了线程不安全的问题   
   解决办法: 加锁,但是效率带来了降低
